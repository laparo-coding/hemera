# Contracts: Test Coverage (Spec 028)

This feature does not add public product-facing HTTP endpoints.

Instead, Phase 1 defines planning contracts for how coverage improvement and CI gating are expected
to behave.

## Artifacts

- `coverage-gates.md` — defines the planning contract for coverage measurement, critical-area
  prioritization, and CI gate rollout

## Contract Intent

These contracts are used to:

- keep Feature 028 measurable
- prevent coverage work from collapsing into vanity metrics
- align implementation tasks with CI-verifiable quality outcomes
